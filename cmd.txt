docker pull: Загрузка образа из Docker Hub или другого репозитория образов.
docker pull <имя_образа>

docker build: Создание образа из Dockerfile (инструкции по настройке образа).
docker build -t <имя_образа> <путь_к_Dockerfile>

docker run: Запуск контейнера на основе образа.
docker run <опции> <имя_образа>

docker ps: Отображение активных контейнеров.
docker ps

docker exec: Запуск команды внутри работающего контейнера.
docker exec <опции> <идентификатор_контейнера> <команда>

docker stop: Остановка контейнера.
docker stop <идентификатор_контейнерa

docker rm: Удаление контейнера.
docker rm <идентификатор_контейнера

docker rmi: Удаление образа.
docker rmi <имя_образа>

docker logs: Просмотр логов контейнера.
docker logs <идентификатор_контейнера>

docker network: Управление сетями Docker.
docker network <подкоманда> <опции>

docker-compose: Используется для управления многоконтейнерными приложениями с помощью файла конфигурации.
docker-compose <команда> <опции>

docker info: Отображение информации о Docker системе.
docker info
Это всего лишь несколько основных команд. Docker предоставляет множество других команд и опций для управления контейнерами, образами, сетями и томами данных. Для получения более подробной информации, вы можете воспользоваться встроенной справкой:


docker --help
или

docker <команда> --help
